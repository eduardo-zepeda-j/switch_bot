Arquitectura de Switch_bot

Procesamiento multicanal, guion previo, notas en manual/app y motor RAG con AWS Bedrock para marcadores EDL.

Frecuencia del Sistema Configurada: 60 fps 30 fps 29.97 fps Frame Time: 33.33 ms Alineación SMPTE: TOD (Time of Day)
Proceso 1: Captura Multicanal Asíncrona & Ingesta Tripartita de Marcadores
Búfer acoplado a 30 FPS + Sincronización de Contexto de Producción
1. Video & Audio Stream • 4x CSD/DSHOW Video Feeds
• Microphones PCM Stream
• Inferencia MediaPipe + VAD
2. Guión Previo Cargado • PDF / Markdown / JSON Script
• Estructura de bloques y cues
• Mapeo de personajes y escenas
3. App / GUI Live Control • Botones de Notas Rápidas (App)
• Entrada Manual (Keyboard/UI)
• Marcador por Prompt de IA
Memoria Compartida (multiprocessing.Queue + Thread Event Loop)
Aislamiento total para prevenir bloqueos de frame time. Enruta video a inferencia y prompts/notas al pipeline EDL.
Proceso 2: Pre-procesamiento de Contexto & Visión Artificial
Gaze Tracking + Voice Activity Detection (VAD) + Parser de Guión en Memoria
Enriquecimiento Semántico & Marcadores IA: AWS Bedrock API
• Ingesta de Guión Vectorizado: Titan Embeddings V2 genera base RAG con la escaleta del programa.
• Análisis Contextual & Prompts: Claude 3.5 Sonnet / Haiku compara el audio en vivo con el guión o procesa prompts manuales (ej: "Marca esta discrepancia respecto al guión").
• Generación de Comentario EDL: Produce notas enriquecidas formateadas para CMX 3600 con categoría, color y metadatos.

1. Pipeline ATEM (Hardware)
Conmutación SDI/HDMI:
Comandos TCP asíncronos a entrada física mapeada vía PyAtemMax.
Arquitectura Multihilo:
Worker dedicado para sockets de control ATEM ISO sin bloquear la UI.
Feedback Visual:
Tally QFrame actualizado cada 33.33 ms.
2. Pipeline OBS MCP (Software)
Cliente Protocolo MCP:
Envío de eventos JSON/MCP a OBS Studio por WebSockets/SSE.
Ajuste Dinámico de Escenas:
Cambio automático a escenas específicas (ej. Escena "Carlos_CloseUp").
Resiliencia de Red:
Manejo de reconexión asíncrona en caso de micro-cortes de red.
3. Pipeline Metadata & Bedrock
Append-Only Log (.jsonl):
Registro en caliente con ID de personaje, timecode y la nota ingresada/generada.
Sintetizador Bedrock (.drp):
bedrock:InvokeModel consolida notas de guión y genera capítulos para NLE.
Compilador .drp Final:
Generación del proyecto para DaVinci Resolve estructurado.
4. Pipeline Multi-Marker EDL Engine
Clasificador de Origen de Nota:
Procesa y categoriza la nota según su fuente en el Timecode SMPTE activo.
Fuentes Soportadas:
• Manual: Notas escritas desde la App.
• IA / Contexto: Coincidencia con Guión cargado / Prompt a Bedrock.
Sintetizador CMX 3600 & Colors:
Formatea comentarios con códigos de color de marcadores EDL:
* FROM CLIP: CAM_02
* |C:Red |M:MANUAL_NOTE - Repetir toma
* |C:Green |M:SCRIPT_MATCH - Bloque 2 Iniciado
* |C:Magenta |M:AI_PROMPT - Error de pronunciación
Escritor de Archivos .edl en Caliente:
Exportación continua de session_markers.edl con actualización atómica en tiempo real.
